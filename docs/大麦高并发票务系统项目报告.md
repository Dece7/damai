# 大麦高并发票务系统项目报告

## 一、项目概述

这是一个**真实还原大麦网高并发抢票场景**的全栈微服务项目，采用 Spring Cloud Alibaba 微服务架构，包含完整的后端微服务和前端 Vue3 管理/用户界面。

**核心业务场景**：演唱会/演出票务在线抢购，涵盖用户注册登录、节目浏览、座位选择、订单创建、支付、退款等完整票务流程。

## 四、微服务详细分析

### 4.1 damai-gateway-service（网关服务）

| 项目 | 说明 |
|------|------|
| **端口** | 6085 |
| **职责** | 统一入口、路由转发、限流、API 访问控制、Kafka 消息采集 |
| **核心组件** | Sentinel 限流、Redis Lua 限流、Kafka API 数据采集、Knife4j 聚合文档 |

**关键类**：
- `ApiRestrictService` - API 访问限制（基于 Redis + Lua 脚本的多级限流规则）
- `RequestValidationFilter` / `ResponseValidationFilter` - 请求/响应拦截
- `RateLimiter` - 基于 Redis 的令牌桶限流
- `TokenService` - JWT Token 验证
- `ApiDataMessageSend` - Kafka 消息发送（API 调用数据采集）

**限流能力**：
- 支持普通规则（固定阈值）和深度规则（时间段 + 阈值的组合）
- 基于 Redis Lua 脚本实现原子性限流操作
- 支持 IP + 用户 ID + API 路径的多维度限流

### 4.2 damai-user-service（用户服务）

| 项目 | 说明 |
|------|------|
| **职责** | 用户注册、登录、验证码、购票人管理、实名认证 |
| **核心设计** | 组合模式注册校验、布隆过滤器、Redis Lua 频次控制 |

**关键类**：
- `UserService` - 用户核心业务
- `UserCaptchaService` - 验证码管理
- `TokenService` - JWT Token 管理
- `TicketUserService` - 购票人信息管理
- `AbstractUserRegisterCheckHandler` - 组合模式：注册前多级校验（用户是否存在、注册次数、验证码）
- `UserBloomFilterInitData` - 布隆过滤器初始化
- `CheckNeedCaptchaOperate` - Redis Lua 脚本判断是否需要验证码
- `RequestCounter` - 请求频次统计

### 4.3 damai-program-service（节目/演出服务 - 核心）

| 项目 | 说明 |
|------|------|
| **职责** | 节目管理、座位管理、票价管理、演出场次、下单策略、缓存管理 |
| **核心设计** | 策略模式下单、布隆过滤器、Elasticsearch 搜索、本地缓存、Redis Stream 消费 |

**关键类**：
- `ProgramService` - 节目核心业务
- `ProgramOrderService` - 节目订单服务
- `SeatService` - 座位管理

**设计模式**：
- **策略模式**（下单 V1-V4）：`ProgramOrderStrategy` 接口 + `ProgramOrderContext` 上下文

- **组合模式**（下单校验）：`AbstractProgramCheckHandler` 抽象类
  - `ProgramBloomFilterCheckHandler` - 布隆过滤器校验
  - `ProgramDetailCheckHandler` - 节目详情校验
  - `ProgramOrderCreateParamCheckHandler` - 下单参数校验
  - `ProgramUserExistCheckHandler` - 用户存在校验
  - `ProgramRecommendCheckHandler` - 推荐校验

**缓存体系**：
- `LocalCacheProgram` / `LocalCacheProgramCategory` 等 - 本地缓存（Caffeine）
- `ProgramCacheResolutionOperate` - Redis 缓存操作
- `ProgramSeatCacheData` - 座位缓存数据管理

**异步处理**：
- `CreateOrderSend` / `CreateOrderMqDomain` - Kafka 下单消息发送
- `DelayOrderCancelSend` - Redisson 延迟队列（订单超时取消）
- `ProgramRedisStreamConsumer` - Redis Stream 消费者
- `ProgramElasticsearchInitData` - Elasticsearch 数据初始化

## 八、核心架构设计模式

### 8.1 高并发抢票流程

```
用户请求 → Gateway 限流(Sentinel/Redis) → 节目服务
    ↓
布隆过滤器校验 → 本地缓存查询 → Redis 缓存查询
    ↓
策略模式下单(V1-V4) → Kafka 消息队列
    ↓
订单服务消费 → Redisson 分布式锁 → 分库分表写入
    ↓
延迟队列(订单超时取消) → 支付服务 → 支付宝支付
```

### 8.2 设计模式应用

| 模式 | 应用场景 | 实现类 |
|------|---------|--------|
| **策略模式** | 下单策略 V1-V4 | `ProgramOrderStrategy` + `ProgramOrderContext` |
| **策略模式** | 支付渠道 | `PayStrategyHandler` + `PayStrategyContext` |
| **组合模式** | 下单校验链 | `AbstractProgramCheckHandler` + 多个实现 |
| **组合模式** | 用户注册校验 | `AbstractUserRegisterCheckHandler` + 多个实现 |
| **组合模式** | 应用初始化 | `InitializeHandler` + 多个实现 |
| **模板方法** | 支付/退款 | `PayStrategyHandler` 基类 |

### 8.3 高并发手段

| 手段 | 实现 |
|------|------|
| **多级缓存** | 本地缓存(Caffeine) + Redis + 布隆过滤器 |
| **限流** | Sentinel 熔断 + Redis Lua 令牌桶 + 多级规则 |
| **异步解耦** | Kafka 消息队列 |
| **延迟队列** | Redisson DelayQueue（订单超时取消） |
| **分布式锁** | Redisson Lock |
| **分库分表** | ShardingSphere（order/pay/program/user 各 2 库） |
| **搜索引擎** | Elasticsearch（节目搜索） |
| **分布式ID** | Baidu UID Generator（Snowflake） |
| **灰度发布** | 自研灰度框架 |

## 九、Feign Client 模块

`damai-server-client` 作为独立模块，定义了各微服务之间的 Feign 调用接口和 DTO/VO：

| Client | 调用目标 | 主要功能 |
|--------|---------|---------|
| `UserClient` | user-service | 用户查询、登录、注册、购票人管理 |
| `ProgramClient` | program-service | 节目查询、座位、票价、下单 |
| `OrderClient` | order-service | 订单创建、支付检查、取消 |
| `PayClient` | pay-service | 支付、退款 |
| `BaseDataClient` | base-data-service | 区域、渠道数据 |
| `CustomizeClient` | customize-service | 规则管理 |
| `JobClient` | admin-service | 定时任务管理 |

每个 Client 都有对应的 `Fallback` 实现（Hystrix 降级）。
